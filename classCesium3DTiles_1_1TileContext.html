Detailed Description

A context in which a Tileset operates.

The context summarizes the information which is needed by a tileset in order to load Tile data. This includes the base URL that a tileset was loaded from, as well as request headers. The data of individual tiles is obtained by resolving the relative URLs that are obtained from the tiles against the base URL of the context.

One tile context is created for each tileset when the initial tileset data is received. When further tiles are loaded or created, they may create additional contexts - for example, for external tilesets that generate a whole new context with a new base URL. Each context is added to the set of contexts of the tileset with Tileset::addContext.

Tilesets that contain terrain tiles may additionally create an ImplicitTilingContext.
