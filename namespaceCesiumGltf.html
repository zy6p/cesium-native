enum CesiumGltf::AccessorViewStatus
strong

Indicates the status of an accessor view.

The AccessorView constructor always completes successfully. However, it may not always reflect the actual content of the Accessor, but instead indicate that its AccessorView::size is 0. This enumeration provides the reason.

Enumerator
Valid 

This accessor is valid and ready to use.

InvalidAccessorIndex 

The accessor index does not refer to a valid accessor.

InvalidBufferViewIndex 

The accessor's bufferView index does not refer to a valid bufferView.

InvalidBufferIndex 

The accessor's bufferView's buffer index does not refer to a valid buffer.

BufferViewTooSmall 

The accessor is too large to fit in its bufferView.

BufferTooSmall 

The accessor's bufferView is too large to fit in its buffer.

WrongSizeT 

The sizeof(T) does not match the accessor's Accessor::computeBytesPerVertex.

InvalidType 

The Accessor:type is invalid.

InvalidComponentType 

The Accessor::componentType is invalid.

◆ ExtensionState

enum CesiumGltf::ExtensionState
strong

The state of a glTF extension.

Enumerator
Enabled 

The extension is enabled.

If a statically-typed class is available for the extension, it will be used. Otherwise the extension will be represented as a CesiumUtility::JsonValue.

JsonOnly 

The extension is enabled but will always be deserialized as a CesiumUtility::JsonValue.

Even if a statically-typed class is available for the extension, it will not be used.

Disabled 

The extension is disabled.

It will not be represented in the loaded model at all.

Write a glTF or glb asset and its associated external images and buffers to multiple files via user provided callback.

Returns
A CesiumGltf::WriteModelResult containing a list of errors, warnings, and the final generated std::vector<std::byte> of the glTF asset (as a GLB or GLTF).
Parameters
modelFinal assembled glTF asset, ready for serialization.
optionsOptions to use for exporting the asset.
filenameFilename of the final glTF / GLB asset or an encountered external CesiumGltf::ImageCesium / CesiumGltf::BufferCesium during the serialization process
writeGLTFCallbackCallback the user must implement. The serializer will invoke this callback everytime it encounters a CesiumGltf::Buffer or CesiumGltf::Image with an external file uri, giving the callback a chance to write the asset to disk. The callback will be called a final time with the provided filename string when its time to serialize the final glb or gltf to disk.

Simialr to CesiumGltf::writeModelAsEmbeddedBytes, with the key variation that a filename will be automatically generated for your CesiumGltf::Buffer or CesiumGltf::Image if no uri is provided but buffer.cesium.data or image.cesium.pixelData is non-empty.

CESIUMGLTFWRITER_API WriteModelResult CesiumGltf::writeModelAsEmbeddedBytes ( const Model &  model,
const WriteModelOptions &  options 
)

Write a glTF or glb asset to a byte vector.

Parameters
modelFinal assembled glTF asset, ready for serialization.
optionsOptions to use for exporting the asset.
Returns
A CesiumGltf::WriteModelResult containing a list of errors, warnings, and the final generated std::vector<std::byte> of the glTF asset (as a GLB or GLTF).

Serializes the provided model object into a byte vector using the provided flags to convert. There are a few special scenarios with serializing CesiumGltf::Buffer and CesiumGltf::Image objects:

  • If GltfExportType::GLB is specified, model.buffers[0].cesium.data will be used as the single binary data storage GLB chunk, so it's the callers responsibility to place all their binary data in model.buffers[0].cesium.data if they want it to be serialized to the GLB.
  • If GltfExportType::GLB is specified, model.buffers[0].uri CANNOT be set or a URIErroneouslyDefined error will be returned.
  • If a CesiumGltf::Buffer or CesiumGltf::Image a contains base64 data uri and its cesium.data or cesium.pixelData vector is non empty, a AmbiguiousDataSource error will be returned.
  • If a CesiumGltf::Buffer contains a base64 data uri and its byteLength is not set, a ByteLengthNotSet error will be returned.
  • If a CesiumGltf::Buffer or CesiumGltf::Image contains an external file uri and its cesium.data or cesium.pixelData vector is empty, a MissingDataSource error will be returned.
  • If a CesiumGltf::Buffer or CesiumGltf::Image contains an external file uri, it will be ignored (use CesiumGltf::writeModelAndExternalFiles for external file support).
